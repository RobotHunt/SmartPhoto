> 影响范围：前端主图结果页、文案编辑面板、反馈组件

## 一、核心新功能：文字编辑保图模式

### 问题背景

之前用户在主图结果页修改文案后触发重生成（`POST /assets/{asset_id}/regenerate`），**整张图都会变**——构图、背景、风格全部重新生成。用户只想改个标题或卖点文字，却丢了满意的画面。

### 新增接口

```
POST /api/v2/assets/{asset_id}/edit-text
```

**作用**：保持图片构图/色调/产品位置不变，仅替换图上的可见文案。

**请求头**：
```
X-App-Key: <app-key>
Idempotency-Key: <optional>
Content-Type: application/json
```

**请求体**：
```json
{
  "copy_blocks": {
    "headline": "新标题文案",
    "supporting": "新副标题",
    "proof_lines": ["新卖点1", "新卖点2"],
    "matrix_lines": ["参数1", "参数2"]
  },
  "instruction": "字体改大一点"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `copy_blocks` | `object` | 是 | 要替换的文案块。只传需要改的 key，不传的保持原样 |
| `copy_blocks.headline` | `string` | 否 | 主标题 |
| `copy_blocks.supporting` | `string` | 否 | 副标题/支撑文案 |
| `copy_blocks.proof_lines` | `string[]` | 否 | 证据/卖点行 |
| `copy_blocks.matrix_lines` | `string[]` | 否 | 参数/矩阵行 |
| `instruction` | `string \| null` | 否 | 附加指令，如"字体改大"、"标题换红色" |

**响应**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "job_id": "xxx",
    "session_id": "xxx",
    "job_type": "edit_asset_text",
    "status": "queued"
  }
}
```

**前置条件**：
- Asset 必须是 `main_gallery` 类型且 `status == "ready"`
- Asset 必须有 `generation_snapshot`（即真实生成过的图，不是 carry forward 的空壳）
- Session 必须有已完成的结果版本（`latest_result_version > 0`）
- 不能有正在运行的生成任务（`40901 job_already_running`）

**错误码**：
- `40001 invalid_request` — asset 不是 main_gallery / 无 generation_snapshot / 无 image_url
- `40002 invalid_session_status` — 结果还没 ready
- `40901 job_already_running` — 有其他生成任务在跑
- `40902 duplicate_idempotency_key` — 重复幂等键

### 与 regenerate 的区别

| | `edit-text`（新） | `regenerate`（已有） |
|---|---|---|
| **触发场景** | 用户只改了文案内容 | 用户要重新生成整张图 |
| **构图保持** | 最大程度保留原图构图/风格 | 完全重新生成 |
| **策略重建** | 不重建策略，直接基于原 prompt 改 | 重建完整策略 + prompt |
| **参考图** | **已生成图**作首位参考 + 商品图辅助 | 仅商品图作参考 |
| **job_type** | `edit_asset_text` | `regenerate_asset` |
| **版本语义** | 一样，`version_no + 1`，其他 slot carry forward | 一样 |

### 前端适配建议

1. **主图结果页的文案编辑面板**：
   - 如果用户只修改了 `copy_blocks`（headline/supporting/proof_lines/matrix_lines），调用 `edit-text`
   - 如果用户写了自由指令（如"换个背景"、"产品放左边"）或改了非文案内容，继续调用 `regenerate`

2. **Job 状态轮询**：
   - `edit_asset_text` 的事件流与 `regenerate_asset` 完全一致：`job_queued → job_started → job_progress → asset_ready → job_succeeded`
   - 前端无需为新 job_type 写新的轮询逻辑

3. **结果展示**：
   - 结果通过 `GET /sessions/{session_id}/results` 正常获取，新版本包含编辑后的图 + 其他 slot 的 carry forward
   - 编辑后的 asset 的 `generation_snapshot.edit_mode == "text_replace"` 可用于 UI 标记"文字编辑版本"

4. **原始 copy_blocks 获取**：
   - 从当前 asset 的 `generation_snapshot.copy_blocks` 中读取原始文案
   - 前端编辑面板用这个作为初始值，用户修改后的值作为 `copy_blocks` 传给 `edit-text`

### generation_snapshot 新增字段

通过 `GET /sessions/{session_id}/results` 返回的 asset 中，如果是文字编辑生成的：

```json
{
  "generation_snapshot": {
    "edit_mode": "text_replace",
    "source_asset_id": "原始asset的ID",
    "source_version_no": 3,
    "text_changes": {
      "source_copy_blocks": { "headline": "旧标题", ... },
      "new_copy_blocks": { "headline": "新标题" },
      "merged_copy_blocks": { "headline": "新标题", "supporting": "保持不变的副标题", ... }
    },
    "reference_image_ids": ["generated:原始assetID", "商品图ID1", "商品图ID2"]
  }
}
```

## 二、版式语法升级（对前端透明）

以下改动不影响前端接口，但会影响生成图的视觉效果：

1. **Layout Recipe**：每个槽位现在有结构化的版式配方（产品位置、标题区、框架感、利益标签数等），生成的图会有更明确的电商版式骨架
2. **平台语法族群**：1688/淘宝 → 强导购框架型；Amazon/TEMU → 合规白底型；小红书 → 生活方式型；抖音 → 缩略图冲击型
3. **保真硬约束 v3**：logo/文字/颜色三重锁定 + prompt 三位置锚定，减少商品本体被改写的情况

这些改动对前端完全透明，无需做任何适配。

## 三、反馈标签扩展

`POST /api/v2/assets/{asset_id}/feedback` 的 `issue_tags` 现在支持更多标签：

### 新增版式问题标签
| 标签 | 含义 |
|------|------|
| `weak_frame_structure` | 框架感不足 |
| `not_platform_native` | 不像平台原生电商图 |
| `too_photographic` | 太像摄影图，缺导购结构 |
| `insufficient_title_zone` | 标题区不够明确 |
| `lack_of_proof_blocks` | 缺证据块/参数展示 |
| `layout_too_empty` | 版面太空 |
| `layout_too_busy` | 版面太拥挤 |

### 新增保真问题标签
| 标签 | 含义 |
|------|------|
| `logo_moved` | logo 位置被改 |
| `product_text_changed` | 产品上的文字被改 |
| `color_shifted` | 颜色偏移/变色 |

### 完整标签集（17 个）
```
deformed, wrong_color, bad_text, wrong_style, platform_violation,
low_fidelity, other, weak_frame_structure, not_platform_native,
too_photographic, insufficient_title_zone, lack_of_proof_blocks,
layout_too_empty, layout_too_busy, logo_moved, product_text_changed,
color_shifted
```

前端反馈组件如果有标签选择器，建议按以下分组展示：

- **商品问题**：`deformed`, `wrong_color`, `logo_moved`, `product_text_changed`, `color_shifted`, `low_fidelity`
- **文案问题**：`bad_text`
- **版式问题**：`weak_frame_structure`, `not_platform_native`, `too_photographic`, `insufficient_title_zone`, `lack_of_proof_blocks`, `layout_too_empty`, `layout_too_busy`
- **风格问题**：`wrong_style`, `platform_violation`
- **其他**：`other`

## 四、策略预览新增字段（信息性）

`GET /sessions/{session_id}/strategy/preview` 返回新增：

```json
{
  "supplementary_view_suggestions": [
    {
      "view_type": "size_comparison",
      "label": "尺寸对比图",
      "description": "与常见物品对比展示产品实际大小",
      "priority": "recommended"
    }
  ]
}
```

这是按品类推荐的"建议补传哪些图"，前端可选择性展示为提示卡片。当前覆盖 15+ 品类。

## 五、PlatformConfig 新增字段

后台管理 `platform_configs` 表新增 `grammar_family` 字段（`VARCHAR(64) NULL`），用于指定平台视觉语法族群。前端管理面板如有平台配置编辑，可暴露此字段为下拉选择：

```
guided_conversion    强导购框架型（1688/淘宝）
structured_retail    结构化零售型（京东/拼多多）
compliance_white     合规白底型（Amazon/TEMU）
lifestyle            生活方式型（小红书/官网）
thumbnail_impact     缩略图冲击型（抖音/TikTok）
brand_free           自由品牌型（自定义站点）
```
