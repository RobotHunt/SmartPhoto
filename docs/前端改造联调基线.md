# 前端改造联调基线

## 1. 当前真相源

- 前端仓库：`/home/wppjkw/SmartPhoto`
- 后端仓库：`/home/wppjkw/smartphoto_backend`
- 联调真相文档：后端 `docs/API_联调指南.md`

当前以后端最新实现为准，前端已经按“纯图片 SaaS + `X-App-Key`”模式收口。

---

## 2. 当前联调口径

- 图片主链路统一走 `/api/v2`
- 后端只认 `service_id + session_id`
- `/api/v2/auth/*`、`/api/v2/account/*`、`/api/v2/guest/*` 已下线
- 浏览器本地联调通过 Vite 代理注入 `X-App-Key`
- 生产级 BFF / 服务端代理不在本轮范围内

本地开发至少需要：

```env
VITE_LOCAL_PROXY_TARGET=http://127.0.0.1:8000
LOCAL_PROXY_X_APP_KEY=replace-with-your-local-app-key
```

说明：

- `LOCAL_PROXY_X_APP_KEY` 只给 `vite.config.ts` 读取
- 浏览器运行时代码不会直接暴露这个 key
- 若配置了 `VITE_LOCAL_PROXY_TARGET` 但没配 `LOCAL_PROXY_X_APP_KEY`，开发启动会直接报错

---

## 3. 当前前端流程

### 3.1 主图链路

当前路由顺序仍保持：

`upload -> analyze -> platform -> generate -> strategy -> result -> payment -> hd-result`

但语义已经调整为：

- `payment` 是“当前版本确认页/过渡页”
- `hd-result` 是“当前版本结果查看页”
- 不再承载支付、额度、登录、认领、归档到账户等旧语义

### 3.2 详情图链路

当前路由顺序仍保持：

`hd-result -> copywriting -> detail-confirm -> detail-result`

其中：

- `detail-result` 直接走真实 `detail-pages/generations/results/download`
- 不再依赖登录、claim、`can_download` 等旧用户字段

---

## 4. 当前已完成的适配

### 4.1 API 层

`client/src/lib/api.ts` 已按纯图片 SaaS 语义收口：

- 不再拼 `Authorization`
- 不再做 token refresh / `auth:logout`
- SSE 不再拼 `?token=...`
- 主图、详情图下载都直接走当前 `/sessions/{id}/download*`
- `authAPI`、`accountAPI`、`pricingAPI`、`claimGuestSession` 已改为显式下线 stub

### 4.2 Session 快照与结果字段

当前前端继续依赖的 session 字段主要是：

- `session_id`
- `status`
- `current_step`
- `selected_platform_ids`
- `active_platform_id`
- `analysis_snapshot`
- `parameter_snapshot`
- `confirmed_copy`
- `strategy_preview`
- `detail_strategy_preview`
- `generation_round`
- `latest_result_version`
- `detail_generation_round`
- `detail_latest_result_version`

不再把这些旧用户字段当成有效驱动：

- `auth_mode`
- `guest_quota_remaining`
- `login_required_actions`
- `can_download`
- `charged_credits`
- `balance_after`
- `pricing_rule_id`

### 4.3 用户页

以下路由当前保留，但都已改成占位兼容页：

- `/login`
- `/account`
- `/history`
- `/project/:id`

占位页只做两件事：

- 说明后端用户体系已下线
- 提供“继续当前会话 / 开始新的创作”两个入口

注意：

- 这些页面当前不代表后端仍支持用户体系
- 本轮不做本地假账户、本地假历史重建

### 4.4 高清与详情图结果页

当前实现已改成：

- `payment` 不再请求钱包、价格、账户额度
- `hd-result` 直接按当前版本展示主图结果并下载 ZIP
- `detail-result` 直接按当前版本展示详情图结果并下载 ZIP
- 页面里不再出现 claim、登录回跳、账户归档等流程

## 6. 当前已知边界

- 这轮只解决本地联调，不覆盖线上安全注入 `X-App-Key`
- 高清链路仍保留旧路由壳，但文案与行为已经改成纯结果查看语义
- “记住品牌风格”仍是前端本地占位，不回传后端
- 若需要恢复账户历史、登录认领或多租户正式前端接入，需要后续单独设计服务端代理方案
