  │
  ├──── (N) orders ──── (1) projects
  │
  └──── (N) verification_codes


四、Account页面数据对应关系

页面区域              │ 数据来源
─────────────────────┼─────────────────────
用户头像/昵称/邮箱     │ users表
使用统计-生成图片      │ COUNT(generatedImages) WHERE userId
使用统计-本月使用      │ COUNT(projects) WHERE userId AND 本月
购买记录列表           │ orders表 JOIN projects JOIN generatedImages
购买记录-查看图片      │ generatedImages WHERE projectId = order.projectId
购买记录-申请退款      │ UPDATE orders SET refundStatus/refundReason/refundNote/refundAt
购买记录-去支付        │ 跳转第三方支付，回调更新 orders.status/paidAt
账户安全-手机号        │ users.phone
账户安全-修改手机号    │ 发送验证码 → verification_codes表 → 验证后更新 users.phone
账户安全-邮箱          │ users.email
账户安全-修改邮箱      │ 发送验证码 → verification_codes表 → 验证后更新 users.email
账户安全-密码状态      │ users.password IS NOT NULL
账户安全-注册时间      │ users.createdAt
账户安全-最近登录      │ users.lastSignedIn


五、退款原因枚举（前端预设选项）

1. 图片质量不满意
2. 生成结果与预期不符
3. 误操作购买
4. 其他（用户在补充说明中填写详情）
