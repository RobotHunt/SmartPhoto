1. users 用户表
┌──────────────┬───────────────┬──────────────────────────────┐

├──────────────┼───────────────┼──────────────────────────────┤

│ phone        │ VARCHAR(20)   │ 手机号（唯一，登录凭证）        │

│ updatedAt    │ TIMESTAMP     │ 更新时间（自动）                 │

三、KV 存储数据映射（临时演示阶段使用）
═══════════════════════════════════════════════════

KV 存储空间与 MySQL 表的对应关系，以及 Key 命名规范：
Key 以用户的 openId 为索引（如 test-user-a），与前端 sessionStorage 中存储的 openId 字段一致。

┌────────────────────────┬──────────────────────────┬────────────────────────────┐
│ KV 存储空间              │ Key 格式                  │ 对应 MySQL 表 / Value 格式    │
├────────────────────────┼──────────────────────────┼────────────────────────────┤
│ smartphoto-users       │ {openId}                 │ users 表 → 单个用户 JSON     │
│                        │ stats:{openId}           │ 聚合查询 → 统计数据 JSON     │
│                        │ notifications:{openId}   │ notifications 表 → 通知数组  │
├────────────────────────┼──────────────────────────┼────────────────────────────┤
│ smartphoto-orders      │ list:{openId}            │ orders 表 → 该用户所有订单    │
│                        │ {orderNo}                │ orders 表 → 单个订单 JSON    │
├────────────────────────┼──────────────────────────┼────────────────────────────┤
│ smartphoto-sessions    │ session_{sessionId}      │ 无对应表 → 登录会话 JSON     │
│                        │                          │ { userId, expireAt }       │
└────────────────────────┴──────────────────────────┴────────────────────────────┘

注意：Key 使用冒号分隔（stats:test-user-a），与旧版下划线格式（stats_user_001）不同。


四、API 接口与数据流对照
═══════════════════════════════════════════════════

所有接口通过 ?uid={openId} 参数区分用户，前端从 sessionStorage.auth_user.openId 自动读取。
现阶段（KV）和后续（MySQL）的 API 路径保持一致，前端不需要改动：

┌──────────────────────────┬────────┬─────────────────────────────────────┐
│ API 路径                   │ 方法    │ 说明                                 │
├──────────────────────────┼────────┼─────────────────────────────────────┤
│ /api/user?uid=xxx         │ GET    │ 获取当前用户信息                      │
│ /api/user/stats?uid=xxx   │ GET    │ 获取使用统计（生成图片数、本月使用）    │
│ /api/orders?uid=xxx       │ GET    │ 获取当前用户的订单列表                 │
│ /api/order/{id}           │ GET    │ 获取单个订单详情                      │
│ /api/order/{id}/pay?uid=  │ POST   │ 发起支付（模拟/真实）                 │
│ /api/order/{id}/refund?uid│ POST   │ 申请退款                             │
│ /api/notifications?uid=   │ GET    │ 获取消息通知列表                      │
│ /api/auth/login           │ POST   │ 登录（手机号+密码 或 手机号+验证码）    │
│ /api/auth/register        │ POST   │ 注册（手机号+密码+验证码）             │
│ /api/auth/logout          │ POST   │ 退出登录                             │
│ /api/auth/send-code       │ POST   │ 发送验证码                            │
│ /api/seed?uid=xxx         │ POST   │ 初始化指定用户种子数据（仅KV阶段使用） │
└──────────────────────────┴────────┴─────────────────────────────────────┘


五、页面 → 数据对应关系
═══════════════════════════════════════════════════

1. 登录页（/auth）
   ├── 密码登录    → POST /api/auth/login { phone, password }
   ├── 验证码登录  → POST /api/auth/send-code → POST /api/auth/login { phone, code }
   └── 注册       → POST /api/auth/send-code → POST /api/auth/register { phone, password, code }

2. 首页（/）
   ├── 用户头像/昵称 → GET /api/user
   └── 登录按钮     → 跳转 /auth

3. 账户页（/account）
   ├── 用户信息     → GET /api/user
   ├── 使用统计     → GET /api/user/stats
   ├── 购买记录     → GET /api/orders
   │   ├── 去支付   → POST /api/order/{id}/pay
   │   ├── 查看图片 → 订单中的 images 字段
   │   └── 申请退款 → POST /api/order/{id}/refund { reason, note }
   ├── 消息通知     → GET /api/notifications
   ├── 账户安全     → GET /api/user（手机号、邮箱、密码状态）
   │   ├── 改手机号 → POST /api/auth/send-code → PUT /api/user { phone }
   │   └── 改邮箱   → POST /api/auth/send-code → PUT /api/user { email }
   └── 退出登录     → POST /api/auth/logout

4. 资产页（/history）
   └── 项目列表     → GET /api/projects（后续实现）

5. 生成流程（/create/*）
   └── 支付环节     → POST /api/orders（创建订单）→ POST /api/order/{id}/pay
