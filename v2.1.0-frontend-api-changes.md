## 一、对外接口变更（前端必读）

### 1.1 详情页结果新增 `copy_blocks` 字段

**接口**: `GET /api/v2/sessions/{id}/detail-pages/results`

**变更**: `panels[]` 中每个 panel 新增 `copy_blocks` 字段，返回图片上实际渲染的文案。

```json
{
  "panels": [{
    "asset_id": "...",
    "panel_id": "...",
    "image_url": "...",
    "copy_blocks": {
      "headline": "高效净化 99.9%",
      "supporting": "360° 环绕净吸",
      "bullet_points": ["宠物家庭适用", "低噪运行"],
      "proof_lines": ["HEPA H13滤网", "CADR 380m³/h"],
      "matrix_lines": ["适用面积 45m²"],
      "cta_line": "立即查看"
    }
  }]
}
```

**用途**: 用户点击详情图"修改"时，用 `copy_blocks` 回显图片上**实际渲染**的文案，解决之前"图片文字和编辑框不一致"的问题。

| 字段 | 类型 | 说明 |
|------|------|------|
| `headline` | string | 主标题 |
| `supporting` | string | 副标题/辅助文案 |
| `bullet_points` | string[] | 要点列表 |
| `proof_lines` | string[] | 证据/参数标签 |
| `matrix_lines` | string[] | 矩阵/网格文案 |
| `cta_line` | string | CTA 行动号召 |

### 1.2 `editAssetText` 现已支持详情图

**接口**: `POST /api/v2/sessions/{id}/assets/{asset_id}/edit-text`

此前仅支持 `main_gallery`，现在同时支持 `main_gallery` 和 `detail_page`。

**请求体**（不变）:
```json
{
  "copy_blocks": {
    "headline": "修改后的标题",
    "supporting": "修改后的副文案",
    "proof_lines": ["新标签1", "新标签2"]
  },
  "instruction": "字体改大一点"
}
```

**效果**: 严格保持原图构图/色调/背景/产品位置不变，仅替换文字区域。

### 1.3 `instruction` 字段增强：主题方向注入

**涉及接口**:
- `POST /api/v2/sessions/{id}/generate`
- `POST /api/v2/sessions/{id}/results/regenerate`
- `POST /api/v2/sessions/{id}/assets/{asset_id}/regenerate`

**变更**: `instruction` 现在支持**主题方向关键词**。当用户输入不能被识别为结构化指令（白底/去文案等）时，会被作为主题方向注入到 prompt 的 goal、subject、constraints、copy_blocks 中。

**示例**:
```json
{ "instruction": "宠物净化" }
```

**注入效果**:
- `goal` → `"围绕'宠物净化'这一主题，{原goal}"`
- `subject` → 追加 `"本图需突出'宠物净化'使用场景及对应卖点"`
- `constraints` → 追加 `"所有可见文案和视觉元素必须围绕'宠物净化'主题展开，不要使用通用化、泛品类措辞"`
- `copy_blocks.headline` → 前置 `【宠物净化】`

**结构化指令（优先匹配）**:

| 类别 | 示例 |
|------|------|
| 背景 | "换白底"、"去掉背景"、"黑底"、"加场景" |
| 文字 | "去掉文案"、"减少文字"、"增加文案" |
| 颜色 | "颜色加深"、"颜色更鲜艳" |
| 构图 | "产品放大"、"换个角度" |
| 风格 | "更简约"、"偏高端" |

### 1.4 `regenerateCopy` 异步流程提醒

`POST /api/v2/sessions/{id}/copy/regenerate` 是**异步接口**，返回 `{job_id, status: "queued"}`。

**正确流程**:
```
POST /copy/regenerate        →  job_id
GET  /copy/regenerate/{job_id} →  轮询拿到 generated_fields
PUT  /copy                   →  将 generated_fields 写入 confirmed_copy（必须！否则不生效）
POST /generate               →  此时才生效
```

## 二、Admin 新增接口

### 品类参数规则 CRUD

**基础路径**: `/api/admin/v1/category-parameter-rules`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/category-parameter-rules` | 列表（支持 `category_slug`、`platform_id` 筛选，分页） |
| GET | `/category-parameter-rules/{rule_id}` | 详情 |
| POST | `/category-parameter-rules` | 创建 |
| PUT | `/category-parameter-rules/{rule_id}` | 更新 |
| POST | `/category-parameter-rules/{rule_id}/archive` | 归档 |
| POST | `/category-parameter-rules/{rule_id}/restore` | 恢复 |

**创建请求体**:
```json
{
  "category_slug": "air_purifier",
  "platform_id": null,
  "core_purchase_parameters": [
    {"key": "cadr", "label": "CADR值", "unit": "m³/h", "priority": 1}
  ],
  "parameter_extraction_hints": ["优先识别净化性能相关参数"],
  "anti_patterns": ["外观形状", "按钮布局方式", "指示灯颜色"],
  "selling_point_themes": [
    {"theme": "宠物家庭适用", "keywords": ["宠物", "毛发", "除味"]}
  ],
  "category_reasoning_hints": "空气净化器是成熟家电品类..."
}
```

系统已为 `air_purifier` 预置种子数据，无需手动创建即可生效。

## 三、内部行为变更（影响效果但接口不变）

| 改动 | 影响范围 | 说明 |
|------|---------|------|
| 参数提取 prompt 重构 | `parameters/extract`、`parameters/complete` | 结合品类知识推理，不再只看图。空气净化器会提取 CADR/滤网等级而非外观形状 |
| Doubao 推理启用 | `parameters/complete` | `reasoning_effort=medium`，补全质量提升 |
| LLM 卖点文案设计 | `strategy`、`generate` | 此前 disabled，现在 Doubao 参与文案设计，文案更有品类区分度 |
| 生图模型切换 | 所有生图 | `gpt-image-2` 替代 `gemini-3.1-flash-image-preview-2k` |
